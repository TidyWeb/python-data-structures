This page covers nested lists.

Its foundation is following one index at a time, from the outside layer towards the value inside.

Python data structures · visual revision guide

Follow the layers.

A nested list is a list containing other lists. Each pair of square brackets takes one deliberate step into the structure.

3questions for every index

01

Start with the organising job

A list can hold whole lists

Nested lists group related values into layers. That makes them useful for tables, grids, timetables, seating plans and grouped orders.

Ordinary list

One level of values

subjects = [ "Maths", "English", "Science" ]

Nested list

Several grouped lists

timetable = [ ["Maths", "English", "Science"], ["History", "Art", "PE"], ["Maths", "Computing", "Music"] ]

The outer list organises the groups. In the timetable, each inner list represents one day and each value inside that inner list represents one subject.

02

Make the structure visible

Outside list → inside list → item

Use the Code Institute restaurant-order example. The variable orders refers to one outer list containing three customer lists.

orders = [ ["pizza", "cola"], # Customer 0 ["burger", "fries"], # Customer 1 ["salad"] # Customer 2 ]
LEVEL 0The variableorders names the complete nested structure.
LEVEL 1The outer listIt contains three customer lists at indices 0, 1 and 2.
LEVEL 2One inner listorders[0] selects Customer 0’s whole order.
LEVEL 3One itemorders[0][1] selects "cola" inside that order.

Do not try to read both indices as one number. Read them as two separate journeys: first choose an inner list, then choose an item inside that list.

03

Take one step inward

One index returns one inner list

print(orders[0])
Printed output['pizza', 'cola']

Start with the complete structure

orders

The outer list contains all three customers.

→

Use outer index 0

orders[0]

The result is Customer 0’s entire inner list.

The first index always acts on the outer list. Here, index 0 means the first customer list—not the first food item.

04

Take the second step

Two indices return one inner value

print(orders[0][1])
Printed outputcola

First index · choose the group

orders[0]

["pizza", "cola"]
[1]

Second index · choose inside it

The second item

"cola"

Read orders[0][1] in plain English: go to the first customer list, then retrieve the second item from that list.

05

Use the same address to make a change

Replace one nested item

Retrieval and assignment begin with the same nested address. The assignment sign and the new value turn the address into a change.

Before · Customer 1

Burger and fries

["burger", "fries"]
orders[1][0] = "salad"

After · Customer 1

Salad and fries

["salad", "fries"]
orders[1][0] = "salad" print(orders)
Printed output[['pizza', 'cola'], ['salad', 'fries'], ['salad']]

Left-to-right intent: go to Customer 1’s first item and give that location the value "salad". Only that one nested position changes.

One more fact: how the line runs

Your left-to-right reading is still the right way to understand what this line is for: go to the index on the left and give it the value on the right. When Python runs the line, it adds one more step: it first works out the value on the right, then stores that result in the position on the left. This matters when the right side is a calculation instead of a plain value.

scores = [[8, 9], [6, 7]] scores[1][0] = scores[1][0] + 1
Right side works out to7Resulting list[[8, 9], [7, 7]]

06

The rule scales

More layers mean more deliberate steps

The lesson also shows three-level structures. The method does not change: each pair of square brackets selects something from the result produced by the brackets before it.

STARTrestaurantThe complete restaurant structure.
INDEX 1[2]Select Table 2.
INDEX 2[1]Select Customer 1 at that table.
INDEX 3[1]Select that customer’s second course: "curry".
print(restaurant[2][1][1])
Printed output from the lesson examplecurry

Never guess the number of indices from the variable name. Draw or inspect the data’s layers, then follow one level at a time.

07

A repeatable reading method

Ask three questions for every index

  • 01

    What collection am I looking at now?
    Begin with the outer variable or the result returned by the previous index.

  • 02

    What does this index select from it?
    Name the group, row, customer, day or item in ordinary language.

  • 03

    What type of thing did that selection return?
    If it returned another list, the next index acts inside that list.

  • 04

    Have I reached the value I need?
    Stop when the result is the intended list or item; add another index only when another layer remains.

Trace it with temporary steps when the brackets feel crowded: store or print the result after the first index, understand that smaller structure, then apply the next index.

customer_zero = orders[0] print(customer_zero) drink = customer_zero[1] print(drink)
Printed output['pizza', 'cola']
cola

08

Apply the LMS timetable scenario

Read and update a school timetable

The structure

One inner list per day

  • Monday: Maths, English, Science
  • Tuesday: History, Art, PE
  • Wednesday: Maths, Computing, Music

Your three jobs

  1. Create the nested list timetable.
  2. Replace Wednesday’s second subject with "Drama", then print the complete timetable.
  3. Print Tuesday’s final subject.
Check the complete timetable exercise
timetable = [ ["Maths", "English", "Science"], ["History", "Art", "PE"], ["Maths", "Computing", "Music"] ] timetable[2][1] = "Drama" print(timetable) print(timetable[1][2])
Printed output[['Maths', 'English', 'Science'], ['History', 'Art', 'PE'], ['Maths', 'Drama', 'Music']]
PE

Wednesday is outer index 2; its second subject is inner index 1. Tuesday is outer index 1; its last subject is inner index 2.

Transfer question: if you print only timetable[1], do you expect one subject or a complete day?

Reveal the transfer answer

A complete day: ['History', 'Art', 'PE']. One index selects one inner list. A second index is needed to retrieve one subject inside it.

09

Recognise what went wrong

Common nested-list traps

Expected one item

print(orders[0]) Actual: ['pizza', 'cola']

One index selected the complete inner list. Add the inner item’s index only if one value is required.

Reversed the two jobs

orders[1][0] expected "cola" Actual: "burger"

The first index chooses Customer 1. Only then does index 0 choose that customer’s first item.

Counted from one

timetable[3][2] Actual: IndexError

Three day lists use outer indices 0, 1 and 2. There is no outer index 3.

Added one index too many

orders[0][1][0] Actual: returns "c", the first character of "cola"

After two indices the result is already a string. Another index acts on that string, not on the original nested list.

10

Before moving on

Quick recall

  1. What makes a list “nested”?
  2. What does the first index act on?
  3. What does orders[0] return in the worked example?
  4. How would you read orders[0][1] in ordinary language?
  5. What visible syntax turns nested retrieval into nested assignment?
  6. Why might a three-level structure require three indices?
Reveal the quick-recall answers
  1. It contains another list—or several lists—as its elements.
  2. The first index acts on the outer list.
  3. Customer 0’s whole inner list: ['pizza', 'cola'].
  4. Choose Customer 0’s list, then retrieve its second item.
  5. An assignment sign followed by the new value, such as = "salad".
  6. Each index moves through one layer: for example table, then customer, then course.

The durable mental model: every index acts on the result to its left. Follow the structure one layer at a time instead of trying to decode all the brackets at once.
