Python collections · Visual revision guide

One name.
Many values.

A Python list gives a group of values one shared home. The important skill is understanding how Python numbers, retrieves, changes and extends the items inside that home.

4core list moves

01

Start with the problem

Why use a list?

A list is useful when several values belong to the same collection and need to be handled together.

Without a list

Every value needs a separate name

student_1 = "Ayaan" student_2 = "Mei" student_3 = "Olivia" student_4 = "Jamal"

Python has no single object representing the whole student group.

With a list

One name describes the collection

student_names = [ "Ayaan", "Mei", "Olivia", "Jamal" ]

student_names represents the collection; each name remains an individual item.

Plain-English definition: a list is an ordered, changeable container that stores several values under one variable name.

02

Read the punctuation

The anatomy of a list

Square brackets mark the outer boundary. Commas divide one item from the next. The equals sign gives the completed list a variable name.

The name

The whole collection

student_names
→

The stored values

Items inside brackets

["Ayaan", "Mei", "Olivia", "Jamal"]

[ square brackets ]

Hold the collection

They show where the list begins and ends.

, commas

Separate the items

Each comma marks a boundary between values.

Mixed types

Values need not match

One list can hold strings, integers, floats and Booleans.

03

Find one item

Python counts from zero

An index is the numbered position Python uses to locate an item. The first item is at index 0, not index 1.

index 0
"Ayaan"
index -4
index 1
"Mei"
index -3
index 2
"Olivia"
index -2
index 3
"Jamal"
index -1

The upper row counts forwards from the beginning. The lower row counts backwards from the end. Colour reinforces that distinction. On a narrow screen, scroll this map sideways.

From the beginning

Retrieve the second item

student_names[1]

Human place: second. Python index: 1.

From the end

Retrieve the final item

student_names[-1]

Negative index -1 always points to the final item.

Ask yourself: “Am I describing the human position, or writing the Python index?”

04

Change what is stored

Lists are mutable

Mutable means the contents can change after creation. Identify one position, then assign its replacement.

Before

Index 2 contains Olivia

["Ayaan", "Mei", "Olivia", "Jamal"]
→

Replacement

Index 2 becomes Riley

student_names[2] = "Riley"

Read it left to right: go to index 2 inside student_names, then replace its value with "Riley".

One more fact: how the line runs

Your left-to-right reading is still the right way to understand what this line is for: go to the index on the left and give it the value on the right. When Python runs the line, it adds one more step: it first works out the value on the right, then stores that result in the position on the left. This matters when the right side is a calculation instead of a plain value.

student_names[2] = student_names[2] + " Chen"
Right side works out to"Riley Chen"Resulting list["Ayaan", "Mei", "Riley Chen", "Jamal"]

05

Grow the collection

Add one item with .append()

The dot connects an object with an operation it can perform. Here, the list is asked to add one value at its end.

The object

Which list changes?

student_names
·

The operation

What should it do?

append("Sophia")
student_names.append("Sophia") # The same list now has one more item.

append() changes the existing list. It does not create a second list.

06

Put the moves together

Four list moves, then check

MOVE 1CreatePut values inside square brackets.
MOVE 2RetrieveUse an index to read one item.
MOVE 3ReplaceAssign to an indexed position.
MOVE 4AppendAdd one item at the end.

Checking is not a fifth list move. Printing the final list is the habit that lets you verify what the four moves produced.

student_names = ["Ayaan", "Mei", "Olivia", "Jamal"] print(student_names[1]) # retrieve second student_names[2] = "Riley" # replace third student_names.append("Sophia") # add at end print(student_names[-1]) # retrieve final print(student_names) # inspect list

07

Debug the thought first

Common traps

  • 01

    Second item ≠ index 2.
    The second item is at index 1.

  • 02

    Index -1 is not first.
    It identifies the final item.

  • 03

    Retrieval does not change a list.
    student_names[1] only reads.

  • 04

    Replacement needs =.
    Assign the new value to its index.

  • 05

    append() adds at the end.
    It does not insert in the middle.

  • 06

    Strings need quotation marks.
    Without them, Python expects a variable.

08

Transfer the pattern

Try it with tools

Starting collection

tools = [ "pen", "notebook", "ruler" ]

The data changed; the list operations did not.

Your questions

Write one expression or statement for each job

  1. Retrieve "notebook".
  2. Retrieve "ruler" from the end.
  3. Replace "pen" with "pencil".
  4. Add "eraser" at the end.
Check your answers after you have attempted all four
  1. tools[1] retrieves the second item, "notebook".
  2. tools[-1] retrieves the final item, "ruler".
  3. tools[0] = "pencil" replaces the first item.
  4. tools.append("eraser") adds one new final item.
tools = ["pen", "notebook", "ruler"] print(tools[1]) print(tools[-1]) tools[0] = "pencil" tools.append("eraser") print(tools)

Final list: ["pencil", "notebook", "ruler", "eraser"]

09

Before moving on

Quick recall

  • 01

    A list stores several values under one name.

  • 02

    Square brackets contain the list and identify indexed positions.

  • 03

    Index 0 is first; index -1 is final.

  • 04

    Lists are mutable, so indexed values can be replaced.

  • 05

    .append(value) adds at the end of the existing list.

  • 06

    Predict, run, then inspect what changed.

The durable mental model: create the collection, locate an item by index, then decide whether you are reading it, replacing it or adding something new.
