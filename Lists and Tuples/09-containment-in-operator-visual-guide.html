This page covers in and not in.

They ask a membership question and return one Boolean answer.

Python data structures · visual revision guide

Ask what is inside.

The in operator checks whether something belongs to a collection. Its answer is always the Boolean value True or False.

2possible answers

01

Start with the question

Does this collection contain it?

Containment is a yes-or-no test. It lets a program check existing data before deciding what should happen next.

What to find

The candidate element

"Thor"
→

Where to look

The collection

in avengers
avengers = ["Thor", "Hulk", "Black Widow"] print("Thor" in avengers)
Printed outputTrue

02

Python answers the question

The result is a Boolean

Found

True

The candidate element belongs to the collection being searched.

Not found

False

The candidate element does not belong to that collection.

in does not retrieve the element. It reports whether the membership statement is true or false.

03

The same question works broadly

Lists, tuples, sets and strings

ExpressionWhat Python checksResult
"apple" in ["apple", "pear"]List elementsTrue
3 in (1, 2, 3)Tuple elementsTrue
"a" in {"a", "b"}Set elementsTrue
"lo" in "hello"Characters or a continuous substringTrue

For these collections, Python checks their contained values. Dictionaries are the important exception explored next.

04

A special dictionary rule

Plain in checks dictionary keys

data = {"name": "Lina", "age": 30} print("name" in data) print("Lina" in data)
Printed outputTrue
False
Key search

"name" in data

"name" is a key, so the result is True.

Not a key

"Lina" in data

"Lina" is a value, so plain dictionary containment returns False.

05

State where Python should search

Use .values() for dictionary values

print("Lina" in data.values())
Printed outputTrue

Keep the two questions separate: candidate in dictionary checks keys; candidate in dictionary.values() checks values.

06

Use the Boolean to control a route

Containment can become an if condition

grocery_list = ["milk", "eggs", "apples", "bread"] if "apples" in grocery_list: print("You already have apples!") else: print("Add apples to your list.")
Printed outputYou already have apples!
QUESTIONAre apples in the list?The expression produces a Boolean.
TRUE ROUTEAlready presentRun the indented if action.
FALSE ROUTENot presentSkip the if action.
ALTERNATIVERun elseTell the learner to add apples.

07

Reverse the membership question

not in checks what is missing

phone_numbers = ["123-456-7890", "555-111-2222"] new_number = "999-888-7777" if new_number not in phone_numbers: phone_numbers.append(new_number) print(phone_numbers)
Printed output['123-456-7890', '555-111-2222', '999-888-7777']

not in is also a Boolean question: “Is this candidate absent from the collection?” Here the answer is True, so Python appends the new number.

08

Apply the LMS airport scenario

Check a scanned item

The data

Restricted items

Use "water bottle", "laptop" and "umbrella". The scanner flags "laptop".

The decision

  1. Create the restricted-items list.
  2. Store the scanned item.
  3. If it is not restricted, print "No issues with this item."
  4. Otherwise, request removal for inspection.
Check the airport-security exercise
restricted_items = [ "water bottle", "laptop", "umbrella" ] scanned_item = "laptop" if scanned_item not in restricted_items: print("No issues with this item.") else: print("Please remove this item for inspection.")
Printed outputPlease remove this item for inspection.

"laptop" is present, so scanned_item not in restricted_items is False. Python therefore runs the else branch.

09

Recognise the wrong mental model

Common containment traps

Expected the element back

"Thor" in avengersActual: True

Containment returns a Boolean, not the matching element.

Searched a dictionary value directly

"Lina" in dataActual: False

Plain dictionary containment checks keys. Use data.values().

Reversed the expression

grocery_list in "apples"Actual: TypeError

Put the candidate on the left and the collection to search on the right.

Misread not in

"Thor" not in avengersActual: False

The statement is false because Thor is present.

Quick recall

Reveal the quick-recall answers
  1. True or False.
  2. Dictionary keys.
  3. Use candidate in dictionary.values().
  4. Whether the candidate is absent from the collection.
  5. Its Boolean result lets Python select which branch should run.

The durable mental model: read candidate in collection as one yes-or-no membership question. With dictionaries, decide explicitly whether “membership” means keys or values.
