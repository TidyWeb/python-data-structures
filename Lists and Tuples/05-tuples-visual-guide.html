This page covers tuples.

Its foundation is the difference between a collection designed to change and one designed to remain fixed.

Python data structures · visual revision guide

Keep a collection fixed.

A tuple stores several ordered values, much like a list. Its defining promise is different: after the tuple is created, its items cannot be replaced, added or removed.

1defining difference

01

Start with the decision

Will this collection need to change?

A list is designed to change. A tuple is designed to remain fixed.

List · changeable

Use square brackets

fruit = ["apple", "banana", "cherry"]

Items can later be replaced, added or removed.

Tuple · fixed

Use parentheses

fruit = ("apple", "banana", "cherry")

The three stored items remain the same after creation.

Immutable means “cannot be changed after creation.” A tuple is an immutable collection.

02

Read the punctuation

The anatomy of a tuple

The LMS lesson distinguishes tuples from lists through their outer punctuation. Parentheses surround the tuple; commas separate its items.

( parentheses )

Mark the fixed collection

fruit = ( ... )
→

, commas

Separate one item from the next

"apple", "banana", "cherry"
fruit = ( "apple", "banana", "cherry" ) print(fruit)
Printed output('apple', 'banana', 'cherry')

Notice the real output: Python prints the tuple with parentheses. That unfamiliar shape confirms that Python has stored a tuple rather than a list.

03

A familiar operation still works

Read an item by its index

Tuples are ordered and zero-indexed, just like lists. The first item is at index 0, so the third item is at index 2.

Index 0 · index -3

"apple"

The first item from the beginning.

Index 1 · index -2

"banana"

The second item from the beginning.

Index 2 · index -1

"cherry"

The third and final item.

print(fruit[2])
Printed outputcherry

Reading is allowed. Immutability prevents changing the stored items; it does not prevent looking at them.

04

Test the defining rule

Try to replace one item

The line below would replace an item successfully in a list. With a tuple, Python refuses the assignment.

fruit[1] = "orange"
Python raisesTypeError: 'tuple' object does not support item assignment

The tuple remains unchanged: ('apple', 'banana', 'cherry')

This failure is the proof. Python found index 1, but the tuple does not permit a new value to be stored there.

05

Let the error teach you

Translate the TypeError

The message looks technical, but it names the exact rule that was broken.

TypeError

The operation does not fit this data type

Python is not saying the tuple disappeared. It is saying this attempted kind of operation is invalid for a tuple.

'tuple' object

The object Python inspected

The quotation marks are part of Python’s error representation. They identify the type as tuple.

does not support item assignment

An item cannot receive a replacement

Assignment means trying to store a new value at an indexed position.

Plain-English translation: “You tried to replace one item, but this collection is a tuple and tuples do not allow that change.”

06

Match the structure to the job

Use a tuple for fixed data

Calendar facts

Days of the week

The expected set and order of weekday names should not be accidentally altered.

Position facts

Fixed coordinates

A coordinate can travel through a program as one ordered pair whose values are meant to stay together.

Recorded facts

Measurements already taken

Historical readings may need to be inspected without being rewritten accidentally.

The useful question is not “Are tuples better?” Ask: “Should this particular collection be allowed to change?”

A smaller advantage

Tuples can also be slightly faster

The LMS notes a small performance benefit. For this lesson, the important reason to choose a tuple is clearer intent and protection from accidental modification.

07

Ask about repetition

Count matching values with .count()

.count(value) returns how many times that value occurs. It reads the tuple and leaves it unchanged.

temperatures = (25, 28, 32, 33, 32, 17, 15) print(temperatures.count(32))
Printed output2

Why 2? The temperature 32 appears twice: once at index 2 and once at index 4.

08

Ask about location

Find the first position with .index()

.index(value) returns the zero-based index of the first matching value. It returns a position, not the value itself.

print(temperatures.index(17))
Printed output5

Human position versus Python index: 17 is the sixth stored temperature, so its zero-based index is 5.

09

The missing methods express the rule

Tuple methods do not change items

OperationAvailable for a tuple?Reason
.count(value)YesIt only reports how many matches exist.
.index(value)YesIt only reports the first matching position.
.append(value)NoAppending would add an item and change the tuple.
.remove(value)NoRemoving would change which items are stored.
.pop()NoPopping would remove and return an item.

The pattern: tuple methods may inspect the fixed collection. Methods whose job is to modify a collection do not exist for tuples.

Use the LMS weather scenario

Track one week of high temperatures

The readings are treated as a completed historical record. That is why the exercise uses a tuple rather than a list.

Seven daily highs in °C

Transfer question: if these temperatures were forecasts that must be revised during the week, would a list or tuple better express that job?

Reveal the transfer answer

A list would better express revisable forecasts because its items may be changed. A tuple better expresses the completed record used in this exercise.

Before moving on

Quick recall

  1. What does immutable mean?
  2. Which outer punctuation distinguishes the lesson’s tuple examples from lists?
  3. Can tuple items still be retrieved by index?
  4. What does .count(value) return?
  5. What does .index(value) return?
  6. Why do tuples have no .append(), .remove() or .pop() methods?
Reveal the quick-recall answers
  1. Immutable means the stored items cannot be changed after the collection is created.
  2. Tuples use parentheses (); lists use square brackets [].
  3. Yes. Tuples are ordered and zero-indexed like lists.
  4. It returns how many times the supplied value occurs.
  5. It returns the zero-based index of the first matching value.
  6. Those methods would modify the collection, which conflicts with tuple immutability.

The durable mental model: a tuple does not prevent you from reading or inspecting its values. It prevents the collection itself from being rewritten.
