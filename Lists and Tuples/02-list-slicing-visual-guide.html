It builds directly on the indexing taught in Lesson 01.

Indexing retrieves one item. Slicing retrieves a run of items by describing where that run starts, where it stops and—when needed—how it moves.

3slice instructions

01

Start with the job

Why slice a list?

Slicing lets you work with a section of a list without retrieving each item separately.

Retrieve one

One item · indexing

Ask for one position

numbers[2]
Python returns"two"
Retrieve a section

Several items · slicing

Ask for a range

numbers[2:6]
Python returns["two", "three", "four", "five"]
numbers = [ "zero", "one", "two", "three", "four", "five", "six", "seven", "eight", "nine" ]

Square brackets define the list. The opening [ and closing ] tell Python that the values between them belong to one list. Commas separate one list item from the next.

Quotation marks do a different job: they define a value as text. The example above stores words, so it uses strings such as "one". A temperature list stores integers, so it uses numbers without quotation marks: [21, 22, 20].

This same ten-item list runs unchanged through Sections 01–07.

Plain-English definition: a slice is a selected section of a sequence. Python writes its instructions between square brackets and separates them with colons.

02

The idea everything else uses

Start included. Stop excluded.

numbers[2:6] begins at index 2 and travels towards index 6, but stops before taking the item at index 6.

What does the colon actually mean?

The colon is a boundary separator. Read it as: “start here : stop before there.” It does not mean “print the number after me.” It separates the starting boundary from the stopping boundary.

2:6 therefore means: include index 2, then take items up to—but not including—index 6.

03

Let Python supply a boundary

Omit the start or the stop

A blank side of the colon means “use the natural boundary of the list.” The colon remains in place because it still separates the start instruction from the stop instruction.

  • 2:6

    TAKE 2, 3, 4, 5 │ STOP BEFORE 6
    Index 2 is included. Index 6 is not.

  • :4

    TAKE 0, 1, 2, 3 │ STOP BEFORE 4
    Python supplies index 0 as the starting boundary.

  • 5:

    TAKE 5, 6, 7, 8, 9 │ THEN END
    Index 5 is included. Python supplies the end of this list as the stopping boundary.

  • :

    Nothing is written; nothing is lost.
    The blank side tells Python to use the list’s beginning or end.

No start written

Begin at the beginning

numbers[:4]

Take indexes 0 to 3. Stop before index 4.

Result: ["zero", "one", "two", "three"]

No stop written

Continue to the end

numbers[5:]

Begin at index 5 and include every remaining item.

Result: ["five", "six", "seven", "eight", "nine"]

Do not read a blank as “nothing.” Before the colon, a blank means “from the beginning.” After the colon, a blank means “to the end.” Remember that 5: includes the item at index 5: it means “from 5 onward,” not “strictly after 5.”

04

Use the nearest end

Slice with negative indexes

Negative indexes count backwards from the end: -1 is the final item, -2 is the second-to-last, and so on.

Instruction

Start fourth-to-last

numbers[-4:-1]
→

Result

Stop before the final item

["six", "seven", "eight"]

The stop rule has not changed. Index -1 is the stopping boundary, so "nine" is excluded.

Practical shortcut: when you want the last few items, a slice such as messages[-5:] is often easier than calculating a positive start index.

05

Control the movement

Add a step—or reverse direction

The optional third instruction is the step, also called the stride. It says how far Python moves between selected items.

Why does “the whole list in steps of two” use two colons?

The full pattern has three instruction slots: [start : stop : step]. The colons are separators that keep those slots in the correct order.

pattern: [START:STOP:STEP] example: [blank:blank:2] start blank → beginning stop blank → end step 2 → jump two

With the spaces removed, [ : : 2 ] becomes [::2]. The two colons preserve the blank start and blank stop slots so that 2 is understood as the step.

Whole-list trace: numbers[::2] → TAKE indexes 0, 2, 4, 6, 8

Start

2

Begin by taking index 2.

Stop

8

Finish before index 8.

Step

2

Move two indexes each time.

numbers[2:8:2] # indexes taken: 2, 4, 6 # result: ["two", "four", "six"]
Retrieve

Every second item

Leave both bounds blank

numbers[::2]

Begin at the start, continue to the end and move two places each time.

Python returns["zero", "two", "four", "six", "eight"]
Retrieve

Reverse the list

Move backwards by one

numbers[::-1]

A step of -1 changes the direction.

Python returns["nine", "eight", "seven", "six", "five", "four", "three", "two", "one", "zero"]

Read three-part slices in this order: start, stop, step. The second colon is needed even when start and stop are blank.

06

Use a slice as a destination

Replace several items together

Change the list

So far, a slice has appeared on its own to retrieve values. Put a slice on the left of = and it becomes the section Python must replace.

Before · original state

Indexes 1, 2 and 3 contain words

["zero", "one", "two", "three", "four", "five", "six", "seven", "eight", "nine"]
numbers[1:4] = [20, 30, 40]

After · changed state

Those three positions contain integers

["zero", 20, 30, 40, "four", "five", "six", "seven", "eight", "nine"]

The assignment itself prints nothing. Print numbers afterwards to see the changed state shown above.

For this lesson’s predictable pattern: replace the selected positions with the same number of new values. Python can resize a list through slice assignment, but that wider behaviour is not required here.

07

Two similar-looking requests behave differently

Indexing can fail; slicing clips safely

The example below contains five items, so its valid positive indexes are 0 to 4.

my_list = [1, 2, 3, 4, 5]

Direct indexing

One missing position is an error

my_list[100]

Python raises IndexError because there is no single item at index 100.

Slicing

An oversized stop is clipped

my_list[3:100]

Python returns the available section: [4, 5]. It does not raise an error.

Why? Indexing promises one exact item. Slicing asks for whatever portion exists inside a range.

Apply every lesson move

Weekly temperature practice

This is the Code Institute lesson’s practice setting. The data changes here deliberately: it tests whether you can transfer the same slicing rules to a new list.

Monday to Sunday

week_temps = [ 21, 22, 20, 19, 24, 23, 22 ]

Before writing each slice, mark the start, stop and step you need.

Part A · retrieve

Predict and write

  1. Tuesday to Thursday.
  2. The weekend using negative indexing.
  3. Sunday back to Monday.
  4. Every second day, beginning Monday.

Part B · modify: replace Thursday and Friday with 18 and 25, then print the updated list.

Check Part A after attempting all four slices
  1. week_temps[1:4] → [22, 20, 19]
  2. week_temps[-2:] → [23, 22]
  3. week_temps[::-1] → [22, 23, 24, 19, 20, 22, 21]
  4. week_temps[::2] → [21, 20, 24, 22]

Part C · diagnose: a learner writes week_temps[-2:-1] for the whole weekend but receives only [23]. What boundary mistake have they made, and what slice retrieves both weekend values?

Check Part C after diagnosing the weekend slice

-1 is an excluded stop boundary, so [-2:-1] takes Saturday but stops before Sunday.

Correction: week_temps[-2:] leaves the stop blank, so Python continues to the end and returns [23, 22].

Check Part B after attempting the replacement
week_temps[3:5] = [18, 25] print(week_temps)

Updated list: [21, 22, 20, 18, 25, 23, 22]

The slice starts at Thursday’s index 3 and stops before Saturday’s index 5, so Thursday and Friday are the two positions replaced.

09

Debug the instruction

Common slicing traps

Stop accidentally included

numbers[2:6] → expected "six" tooActual: ["two", "three", "four", "five"]

The stop boundary is excluded. Use [2:7] if index 6 must be included.

Weekend loses Sunday

week_temps[-2:-1]Actual: [23]

The slice stops before index -1. Leave the stop blank: week_temps[-2:].

One colon cannot hold a step

numbers[:2] → expected every second itemActual: ["zero", "one"]

With one colon, 2 occupies the stop slot. Use [::2] to place it in the step slot.

Reading mistaken for changing

numbers[1:4]Actual: returns a new section; numbers is unchanged

Retrieval has no assignment. To mutate, put the slice left of = and supply replacement values.

10

Before moving on

Quick recall

  1. In list[start:stop:step], which boundary is included and which is excluded?
  2. What do blank start and stop slots mean in numbers[::2]?
  3. Why does numbers[::-1] reverse the list?
  4. What visible syntax distinguishes retrieving a slice from replacing it?
  5. Why can my_list[3:100] succeed when my_list[100] fails?
Reveal the quick-recall answers
  1. The start is included; the stop is excluded.
  2. Blank start means the beginning, blank stop means the end, and step 2 selects every second item.
  3. The blank bounds select the whole list and step -1 moves backwards one position at a time.
  4. A slice on its own retrieves. A slice on the left of = mutates the list.
  5. Indexing demands one exact position; slicing returns the available portion inside a range.

The durable mental model: mark a starting boundary, mark an excluded stopping boundary, then decide whether Python moves one place at a time or follows a different step.
