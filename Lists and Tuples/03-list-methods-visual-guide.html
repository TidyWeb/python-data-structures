It builds on creating, indexing and slicing lists.

Python collections · Visual revision guide

Ask the list
to do a job.

A list method is a built-in operation connected to a particular list with a dot. Some methods change that list. Some return information. One important method—.pop()—does both.

.append().insert().remove().sort().pop()
5active methods

01

Start with the changing world

Why do lists need methods?

Real collections do not stay frozen. New items arrive, old items disappear, order changes, and programs need information about what is stored.

The collection

A fruit display changes

fruits = [ "banana", "apple", "cherry", "strawberry", "cherry" ]

The same five-item starting list runs through Sections 01–06.

The jobs

Manage the collection

  • Add a new fruit.
  • Remove one matching fruit.
  • Put the fruits in order.
  • Take an item out and keep hold of it.

Plain-English definition: a method is an operation that belongs to an object. A list method tells one particular list to perform a list-related job.

02

Read left to right

Object, dot, method, arguments

The punctuation describes who performs the job, which job it performs and what information the job needs.

Names the job

Parentheses ( )

03

Two different destinations

Add at the end or insert at an index

Change · append

Append · one argument

Add one item at the end

fruits.append("blueberry")

.append(item) always places the new item after the current final item.

If applied to the starting list["banana", "apple", "cherry", "strawberry", "cherry", "blueberry"]
Change · insert

Insert · two arguments

Add one item at a position

fruits.insert( 1, "blueberry" )

.insert(index, item) places the item at that index and shifts later items to the right.

Continuing route after insertion["banana", "blueberry", "apple", "cherry", "strawberry", "cherry"]

This lesson’s fruit route uses insertion: placing "blueberry" at index 1 puts it between "banana" and "apple". Nothing is overwritten.

04

Name the value, not its position

.remove() removes the first match

After Section 03 inserts "blueberry", the fruit list still contains two occurrences of "cherry". The method removes only the first matching value.

Change the list

Before · 6 items

Blueberry inserted; two cherries remain

["banana", "blueberry", "apple", "cherry", "strawberry", "cherry"]
fruits.remove("cherry")

After · 5 items

First match removed

["banana", "blueberry", "apple", "strawberry", "cherry"]

Value versus index: .remove("cherry") searches for that value. It does not mean “remove the item at index "cherry".”

Current list: ["banana", "blueberry", "apple", "strawberry", "cherry"]

05

Reorder the existing list

.sort() changes the list in place

In place means the existing list itself changes. Python does not leave the original order untouched.

Change the list

Before sorting

Insertion order

["banana", "blueberry", "apple", "strawberry", "cherry"]
fruits.sort()

After sorting

Alphabetical ascending order

["apple", "banana", "blueberry", "cherry", "strawberry"]

For numbers, ascending means lowest to highest. For these lowercase words, it means alphabetical order.

06

The hybrid method

.pop() removes and returns

Most of the lesson’s methods are easiest to classify as changing the list or returning information. .pop() matters because it does both jobs.

Change + return

popped_fruit = fruits.pop()

Effect 1 · changes the list

The final item is removed

Without an index, .pop() removes "strawberry", the current last item.

["apple", "banana", "blueberry", "cherry"]

Effect 2 · returns a value

The removed item travels

The returned value is assigned to popped_fruit, so it remains available after leaving the list.

popped_fruit == "strawberry"

No index supplied

Remove the final item

colours = ["red", "blue", "green"] removed = colours.pop()
Visible effectsremoved == "green"
colours == ["red", "blue"]

Index supplied

Remove that position

colours = ["red", "blue", "green"] removed = colours.pop(1)
Visible effectsremoved == "blue"
colours == ["red", "green"]

Index 1 means the second item.

Separate experiment: fruits.pop(5) raises IndexError when index 5 does not exist. This experiment is not applied to the continuing worked list.

08

Transfer the method choices

Manage an online-store inventory

This is the Code Institute lesson’s final scenario. The data changes deliberately from fruit strings to stock-count integers.

Starting inventory

inventory = [ 100, 50, 75, 200, 50, 150 ]

Square brackets define the list. These values are integers, so they do not use quotation marks.

Your three jobs

Choose one method for each instruction

  1. Add new stock quantity 30 at the end.
  2. Remove the first occurrence of quantity 50.
  3. Sort the stock quantities in ascending order.

Expected final output: Updated inventory: [30, 50, 75, 100, 150, 200]

Check the method choices after attempting all three jobs
  1. inventory.append(30) adds one new final item.
  2. inventory.remove(50) removes only the first matching 50.
  3. inventory.sort() changes the existing list to ascending order.
inventory = [100, 50, 75, 200, 50, 150] inventory.append(30) inventory.remove(50) inventory.sort() print("Updated inventory:", inventory)

Final list: [30, 50, 75, 100, 150, 200]

Transfer challenge: after sorting the inventory, remove the largest quantity with .pop() and store it in largest_stock. Then explain what changed and what was returned.

Check the transfer challenge
largest_stock = inventory.pop()

Returned value: largest_stock == 200

Changed inventory: [30, 50, 75, 100, 150]

Sorting placed the largest value last; .pop() removed that last item and returned it.

Diagnosis: what goes wrong with inventory = inventory.sort()?

Reveal the diagnosis

.sort() changes the existing list and returns None. Assigning that return value replaces the variable with None. Use inventory.sort() on its own, then print inventory.

09

Ask what the method targets

Common method traps

Value confused with position

fruits.pop("cherry")Actual: TypeError — the pop index must be an integer

Use .remove("cherry") for a value. Use .pop(index) for a position.

Return value confused with the list

inventory = inventory.sort()Actual: inventory becomes None

.sort() mutates in place and returns None. Call it without assignment.

Method named but not called

fruits.sortActual: the list remains unchanged

The parentheses perform the call: fruits.sort().

Missing target

fruits.remove("mango")Actual: ValueError if "mango" is absent

.remove() must find a matching value. For this lesson’s exercises, confirm that the requested value is visibly present first.

10

Before moving on

Quick recall

  1. What does the dot connect in fruits.sort()?
  2. When would you choose .append() instead of .insert()?
  3. Which method targets a value, and which method targets a position?
  4. Why must inventory.sort() not be assigned back to inventory?
  5. What two effects does .pop() have?
Reveal the quick-recall answers
  1. The dot connects a particular list object to the method it must perform.
  2. Use .append() for the end; use .insert(index, item) for a chosen position.
  3. .remove(value) targets a matching value; .pop(index) targets a position.
  4. .sort() changes the list in place and returns None.
  5. .pop() removes an item from the list and returns that removed value.

The durable mental model: before choosing a method, ask two questions: “Do I want to target a value or a position?” and “Should this operation change the list, return information, or do both?”
