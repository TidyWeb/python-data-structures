This page covers sets.

Its foundation is recognising when uniqueness matters more than position or order.

Python data structures · visual revision guide

Keep each value once.

A set stores a collection of unique elements. It is useful for removing repetition and for asking how two groups combine, overlap or differ.

2defining properties

01

Start with the job

Uniqueness matters more than position

A set answers “Which distinct values belong to this group?” It does not answer “What is the third item?”

Property 1 · unique

One copy of each value

Repeated input does not create repeated set elements.

Property 2 · unordered

No dependable display position

Python may print the elements in a different order from the one you typed.

Use a set when the question concerns membership, uniqueness or comparison between groups. Use a list or tuple when sequence and indexed position are part of the meaning.

02

Read the punctuation

Create a set with curly braces

Curly braces surround a populated set. Commas separate one element from the next.

{ curly braces }

Mark the set

hobbies = { ... }
→

Unique elements

Four different hobbies

"reading", "coding", "hiking", "photography"
hobbies = { "reading", "coding", "hiking", "photography" }

An empty set is written set(), not {}. Empty curly braces create an empty dictionary, so Python needs the function form to know that you want a set.

empty_set = set() print(empty_set)
Printed outputset()

03

Elements must be stable

What can a set contain?

The LMS lesson states that set elements must be immutable. Values such as numbers, Booleans, strings and tuples are allowed; mutable lists and dictionaries are not.

Allowed elements

Values that can remain stable

valid = { 42, True, "coding", (3, 7) }
Not allowed

A list cannot be a set element

invalid = {[3, 7]}
One common Python errorTypeError: unhashable type: 'list'

Plain-English translation: Python cannot safely use a changeable list as one distinct set element. Newer Python versions may expand the wording to cannot use 'list' as a set element; the underlying rule is the same.

04

Real output may look rearranged

A set has no promised display order

fruits = { "apple", "banana", "cherry" } print(fruits)
One possible printed output{'apple', 'cherry', 'banana'}

You have not made a mistake if the order differs. Python optimises how a set is stored, so another environment or run may display the same three values in another order. Judge the set by its contents, not by their printed positions.

Practical consequence

Do not retrieve a set element by index

A set is not a sequence, so code such as fruits[0] does not mean “the first fruit.”

05

Test the uniqueness rule

Adding a duplicate changes nothing

.add(value) asks the set to include a value. If the value already belongs to the set, the set remains unchanged.

Before · 3 unique fruits

Cherry already belongs

{"apple", "banana", "cherry"}
fruits.add("cherry")

After · still 3 unique fruits

No second cherry appears

{"apple", "banana", "cherry"}

.add() does not report an error for a duplicate. It simply leaves the set’s unique contents unchanged.

06

Ask the broadest group question

Union: what appears in either set?

first.union(second) creates a new set containing every unique element found in the first set, the second set, or both.

fruits = {"apple", "banana", "cherry"} tropical_fruits = {"banana", "mango", "pineapple"} all_fruits = fruits.union(tropical_fruits) print(all_fruits)
One possible printed output{'apple', 'banana', 'cherry', 'mango', 'pineapple'}

Why only one banana? Banana appears in both source sets, but the union is itself a set and therefore keeps only one copy.

07

Ask about overlap

Intersection: what appears in both?

first.intersection(second) creates a new set containing only elements shared by both source sets.

common_fruits = fruits.intersection(tropical_fruits) print(common_fruits)
Printed output{'banana'}

The overlap contains one value: banana belongs to both fruits and tropical_fruits.

08

Two similar names answer different questions

Difference versus symmetric difference

Directional question

First set only

What belongs to the first set but not the second?

fruits.difference(tropical_fruits)
One possible result{'apple', 'cherry'}
Either side, not both

Exactly one set

What belongs to exactly one of the two sets?

fruits.symmetric_difference(tropical_fruits)
One possible result{'apple', 'cherry', 'mango', 'pineapple'}

The word “symmetric” removes the direction. Ordinary difference begins with the first set. Symmetric difference treats both sides equally and removes their shared overlap.

09

Change the set deliberately

Discard one value or clear everything

Remove one named element

.discard(value)

fruits.discard("banana") print(fruits)
One possible printed output{'apple', 'cherry'}

Remove every element

.clear()

fruits.clear() print(fruits)
Printed outputset()

Why does an emptied set print as set()? Python must distinguish it from {}, which represents an empty dictionary.

10

Choose the question before the method

Set method map

MethodQuestion or jobChanges the original set?
.add(value)Include this value if it is not already present.Yes
.union(other)What belongs to either set?No — returns a new set
.intersection(other)What belongs to both sets?No — returns a new set
.difference(other)What belongs to the first set but not the other?No — returns a new set
.symmetric_difference(other)What belongs to exactly one set?No — returns a new set
.discard(value)Remove this named value.Yes

11

Use the LMS movie-rental scenario

Manage unique movie titles

Two rental days

Starting sets

Day 1: Little Mermaid, Snow White, Avatar

Day 2: Avatar, Titanic, Matrix

Your four jobs

  1. Create both sets.
  2. Create and print the set of all unique rentals.
  3. Create and print the set rented on both days.
  4. Discard "Avatar" from Day 1 and print the changed Day 1 set.
Check the complete movie-rental exercise
rented_movies_day1 = { "Little Mermaid", "Snow White", "Avatar" } rented_movies_day2 = { "Avatar", "Titanic", "Matrix" } all_movies = rented_movies_day1.union(rented_movies_day2) common_movies = rented_movies_day1.intersection(rented_movies_day2) rented_movies_day1.discard("Avatar") print("All unique movies:", all_movies) print("Rented on both days:", common_movies) print("Day 1 after return:", rented_movies_day1)
One possible printed outputAll unique movies: {'Avatar', 'Little Mermaid', 'Titanic', 'Snow White', 'Matrix'}
Rented on both days: {'Avatar'}
Day 1 after return: {'Snow White', 'Little Mermaid'}

Your element order may differ. Check that the membership of each set is correct.

Transfer question: why is "Avatar" shown only once in all_movies even though it was rented on both days?

Reveal the transfer answer

all_movies is a set. Its union includes every title found across the two days, but the set’s uniqueness rule stores only one "Avatar" element.

12

Recognise the behaviour

Common set traps

Expected the typed order

print({"apple", "banana", "cherry"})Actual: the displayed order may vary

A set promises unique membership, not positional order.

Expected a duplicate

{"apple", "apple"}Actual content: {"apple"}

Repeated equal values collapse into one element.

Tried to create an empty set

empty_set = {}Actual type: dictionary

Use empty_set = set().

Tried to retrieve by position

fruits[0]Actual: TypeError — set is not subscriptable

Sets are unordered and do not support indexed retrieval.

Before moving on

Quick recall

  1. What are the two defining properties of a set?
  2. How do you create an empty set?
  3. Why can the printed order differ from the order you typed?
  4. What question does .union() answer?
  5. What question does .intersection() answer?
  6. How does ordinary difference differ from symmetric difference?
  7. Which two methods in this lesson directly remove elements?
Reveal the quick-recall answers
  1. A set contains unique elements and has no promised order.
  2. Use set(); empty curly braces create a dictionary.
  3. Python optimises the set’s internal storage and does not preserve a display sequence.
  4. Which unique elements belong to either set?
  5. Which elements belong to both sets?
  6. Difference keeps first-set-only elements; symmetric difference keeps elements found on exactly one side.
  7. .discard(value) removes one named element; .clear() removes them all.

The durable mental model: a set is about membership, not position. Its methods ask how distinct groups combine, overlap and differ.
