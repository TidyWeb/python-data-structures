This page covers dictionaries

Its foundation is understanding how a key names one stored value.

Python data structures · Visual revision guide

Give information
a useful name.

A dictionary stores information as labelled relationships. Each label is a key; the information found under that label is its value.

2parts in every pair

01

Start with the problem

Why use a dictionary?

Some collections make more sense when each piece of information carries a name.

A list · position answers the question

Which numbered item?

person_details = [ "Nadia", 51, "New York" ]

Index 0 contains "Nadia", but the number 0 does not explain what Nadia represents.

A dictionary · key answers the question

Which named fact?

person = { "first_name": "Nadia", "age": 51, "city": "New York" }

The labels explain the facts: first name, age and city.

The conceptual difference: a list asks, “Which position?” A dictionary asks, “Which named piece of information?”

02

The foundation for the whole lesson

One key points to one value

A key–value pair is one labelled relationship: the key names the information; the value is the information stored under that name.

Key · lookup label

What information do I want?

"first_name"

This is the name Python uses to find the information.

→

Value · stored information

What is stored under that label?

"Nadia"

This is the information returned when that key is requested.

Say it aloud: “The key "first_name" points to the value "Nadia".” Together, those two connected parts are one pair.

key: "first_name"
:
value: "Nadia"
key: "age"
:
value: 51
key: "city"
:
value: "New York"

Important: the key and value are not interchangeable. You supply the key to look up its associated value.

03

Translate the punctuation

Curly brackets hold labelled pairs

person = { "first_name": "Nadia", "age": 51, "city": "New York" }
  • { }

    Curly brackets define the dictionary.
    Everything between them belongs to this one dictionary.

  • :

    A colon connects one key to its value.
    Read it as “points to” or “stores.”

  • ,

    A comma separates one pair from the next.
    It ends the current pair before another begins.

  • " "

    Quotation marks define text.
    The integer 51 needs no quotation marks.

One dictionary, three pairs: person is the whole collection. "age": 51 is one relationship inside it.

Supply a named key

The square brackets do not make this a list. The dictionary already exists. Here, the brackets mean “look inside person using this key.”

05

Use an existing key

Change the value under a label

Change an existing pair

When the key already exists, assignment replaces its associated value. The key stays the same.

Before · age pair

The key points to 51

"age": 51
person["age"] = 52

After · age pair

The same key points to 52

"age": 52

Current dictionary: {"first_name": "Nadia", "age": 52, "city": "New York"}

Look up · no equals sign

Ask for the current value

person["age"]
Python returns52

This line ends after the closing square bracket. It asks a question but stores nothing.

Store or replace · has equals sign

Put a value under the key

person["age"] = 52

The boxed = is the assignment sign. Here it means “store the value on the right in the location named on the left.”

Locationperson["age"]
→ store →
New value52

Read the changed line from left to right: go to the "age" key inside person, then give that location the value 52.

One more fact: how the line runs

Your left-to-right reading is still the right way to understand what this line is for: go to the key on the left and give it the value on the right. When Python runs the line, it adds one more step: it first works out the value on the right, then stores that result under the label on the left. This matters when the right side is a calculation instead of a plain value.

person["age"] = person["age"] + 1
Right side works out to53Resulting dictionary{"first_name": "Nadia", "age": 53, "city": "New York"}

06

Use a key that does not exist yet

Add a new key–value pair

Change · add one pair

The syntax looks like modification. Python decides whether to change or add by checking whether the supplied key already exists.

Before · 3 pairs

No job information

{"first_name": "Nadia", "age": 52, "city": "New York"}
person["job"] = "Engineer"

After · 4 pairs

New label and value added

{"first_name": "Nadia", "age": 52, "city": "New York", "job": "Engineer"}

The deciding question: “Does this key already exist?” Existing key → replace its value. New key → add a new pair.

07

Remove the named relationship

.pop() uses a dictionary key

Change + return

List .pop() targets a position. Dictionary .pop() targets a key. In both cases, the removed value is returned.

Before · 4 pairs

City still exists

{"first_name": "Nadia", "age": 52, "city": "New York", "job": "Engineer"}
removed_city = person.pop("city")

After · 3 pairs

City pair removed

{"first_name": "Nadia", "age": 52, "job": "Engineer"}

Effect 1 · dictionary changes

The pair disappears

Current person{"first_name": "Nadia", "age": 52, "job": "Engineer"}

Effect 2 · value returns

The information remains available

Value stored in removed_city"New York"

Safer form from this lesson: person.pop("country", None) returns None instead of raising an error when the key is absent.

08

Recognise, then look up

A compact dictionary-method map

The lesson does not require the full catalogue from memory. Learn what kind of information each method exposes.

OperationQuestion it answers or job it performsChanges the dictionary?
.keys()What labels are stored?No
.values()What values are stored?No
.items()What key–value pairs are stored?No
.get(key, default)Return a value safely, or a default when the key is absent.No
.pop(key, default)Remove a named pair and return its value.Yes
.clear()Remove every pair.Yes

Keys

Only the labels

person.keys()
Python displaysdict_keys(['first_name', 'age', 'job'])

Values

Only the information

person.values()
Python displaysdict_values(['Nadia', 52, 'Engineer'])

Items

Pairs kept together

person.items()
Python displaysdict_items([('first_name', 'Nadia'), ('age', 52), ('job', 'Engineer')])

Why does Python add dict_keys(...), dict_values(...), or dict_items(...)?

These methods do not return an ordinary list. They return a live view of the dictionary. The unfamiliar wrapper is Python telling you what kind of result you received.

10

Recognise the behaviour

Common dictionary traps

Value used as the lookup

person["Nadia"]Actual: KeyError — "Nadia" is a value, not a key

Supply the label: person["first_name"].

Missing key retrieved directly

person["country"]Actual: KeyError if "country" is absent

Use person.get("country", None) when a missing key is acceptable.

Colon and equals confused

{"age" = 51}Actual: SyntaxError

Inside a dictionary literal, connect a key to its value with a colon: {"age": 51}.

Expected a second age pair

person["age"] = 52Actual: the existing age value is replaced

Dictionary keys are unique. Reusing an existing key changes that pair.

11

Before moving on

Quick recall

  1. In ordinary language, what is a key and what is a value?
  2. What punctuation connects a key to its value inside a dictionary?
  3. How does dictionary lookup differ from list lookup?
  4. What decides whether person["job"] = "Engineer" adds or changes a pair?
  5. What changes and what returns when dictionary .pop() runs?
  6. Which method keeps keys and values together for inspection?
Reveal the quick-recall answers
  1. A key is a lookup label; a value is the information stored under that label. Together they form one pair.
  2. A colon connects each key to its value.
  3. A list uses a numbered index; a dictionary uses a named key.
  4. If the key exists, its value changes. If the key is new, Python adds a new pair.
  5. The named pair is removed from the dictionary and its value is returned.
  6. .items() exposes the dictionary’s key–value pairs.

The durable mental model: a dictionary is not primarily a row of positions. It is a collection of named relationships. Ask for the key; receive the value connected to it.
