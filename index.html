02

See the family resemblance

One question. Four shapes.

A data structure is simply a shape for holding several values together. Every lesson in this collection comes back to the same question: how do I hold more than one value at once, and which shape fits the job? Lists, dictionaries, tuples and sets are the four shapes; the other five lessons cover slicing, methods, two kinds of nesting and containment.

List

Use a list when you need an ordered collection that can change and may contain repeated values.

Dictionary

Use a dictionary when each value needs a descriptive key so you can retrieve it by name.

Tuple

Use a tuple for ordered data that is fixed and should not be changed.

Set

Use a set when values must be unique, order is unimportant, or you need a fast membership check.

With experience, you will start to recognise these needs and choose a shape more intuitively. For now, remember that each structure has a purpose—and that combining shapes can be a powerful way to manage your data.
